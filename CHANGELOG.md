### Changed
- **`DiseaseCategory` union widened** from `"RCC"` to `"RCC" | "ProstateCancer"` in `lib/prompts/disease-registry.ts`. The TypeScript exhaustiveness check via `assertUnreachable()` now requires every dispatcher / consumer to handle both literals; the `case "ProstateCancer":` branch is added to both system-prompt and user-prompt switches in `lib/prompts/structured-report-prompt.ts`. (SPEC-PROSTATE-001)
- **API zod schema widened** ŌĆö `app/api/structured-report/generate/route.ts` `diseaseCategory` field changed from `z.literal("RCC")` to `z.union([z.literal("RCC"), z.literal("ProstateCancer")])`. - **`StructuredReportClient` state model refactored** to a discriminated union (`{ kind: "RCC"; input: RccStructuredInput } | { kind: "ProstateCancer"; input: ProstateStructuredInput }`) so the form, serializer, and minimum-fields validator are all narrowed by TypeScript on every read. The render path branches on `formState.kind` to mount either `<RccStructuredForm>` or `<ProstateStructuredForm>`; per-disease error copy is surfaced when the minimum-fields gate fails. Adding a future disease becomes a compile-error checklist rather than a runtime mismatch. (SPEC-PROSTATE-001)
